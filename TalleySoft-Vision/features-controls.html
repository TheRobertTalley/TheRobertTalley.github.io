Project control map

One page for headset controls, map operations, ATAK-style markers, team tracks, and tactical overlays.

Headset Controls

  • Right-hand dominant mode is default.
  • Left-hand mode mirrors left/right controls and docked HUD locations.
  • Palm toward headset plus index pinch shows or enlarges a quadrant item.
  • Palm away plus index pinch shrinks or hides a quadrant item.
  • Lower-right is weapon camera only: hidden, HUD, center, full.
  • Palm-up pinky pinch cycles weapon camera modes.

Thermal

  • Lower-left controls thermal visibility.
  • Center fusion supports saved near, medium, and far profiles.
  • Calibration covers position, rotation, scale, FOV, crop, warp, distortion, opacity, palette, timing, and distance.
  • Modes are Balanced, Target, Detail, and Manual.
  • Edge palette remains palette 3.

Display Menu

  • Default color theme drives normal HUD and map styling.
  • HUD and compass themes can override their own surfaces.
  • Menu opacity adjusts panel transparency.
  • Clock supports digital, analog, stopwatch, countdown, 24-hour time, and seconds.
  • Reasonable settings expose a reset path on the relevant page.

Map & Mesh

  • Tracks Meshtastic nodes, messages, markers, routes, and compass bearings.
  • Accepts marker text commands such as !target, !location, !route, !lz, and !medical.
  • Accepts ATAK Cursor-on-Target PLI and point events.
  • Breadcrumbs should only be placed when GPS accuracy is 20 yards or better.

ATAK-Style Events

  • !target: target marker with label and decimal coordinates.
  • !location: rally, checkpoint, observation point, or general marker.
  • !route: repeated route points connected by label.
  • !lz: landing-zone marker.
  • !medical: aid or casualty collection marker.
  • Cursor-on-Target PLI and point events are accepted on the configured TAK multicast path.

Web Operations

  • The Vision website is for map operations, marker setup, team tracking, and tactical status.
  • Marker buttons add local preview events without requiring a headset connection.
  • The page accepts live telemetry snapshots over a local WebSocket bridge.
  • OpenStreetMap tiles provide the base map, with marker and route overlays drawn in the browser.
  • Device maintenance tools are intentionally kept off the Vision website.
